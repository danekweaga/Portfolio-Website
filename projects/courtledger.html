Summary

CourtLedger turns scattered records into a searchable history with summary metrics, reporting views and export tools.

My role and contribution

I designed and built the dashboard end to end as the sole developer: authentication, data model, entry and history views, charts, exports and user-scoped access.

Status

Live on Vercel. Source available on GitHub.

Technology

React, TypeScript, Vite, Supabase, PostgreSQL, Recharts.

Problem

Tracking results manually makes it difficult to compare performance, find previous entries or understand where outcomes are coming from.

User workflow

To be documented — describe sign-in, record creation, filtering, reporting and export.

What I personally built

  • Structured record creation, filtering, search and reporting views.
  • Summary metrics and Recharts visualizations.
  • CSV and Excel export plus supported bet-slip OCR.

Architecture or data flow

Supabase authentication with PostgreSQL Row-Level Security keeps records scoped to the signed-in user. The interface separates entry, history and analysis tasks instead of forcing everything into one dashboard state.

To be documented — diagram or more detailed storage and query flow.

Engineering decisions

I used Supabase authentication with PostgreSQL Row-Level Security so records remain scoped to the signed-in user. The interface separates entry, history and analysis tasks rather than collapsing them into a single view.
