Summary

CourtLedger is a React + TypeScript SPA for recording and reviewing personal NBA bets. It uses Supabase Auth and client-side CRUD scoped to the signed-in user, with routes for command center, history, analytics, markets, highlights, and settings.

My role and contribution

Most application code was generated with Cursor and ChatGPT under my direction. I did not independently write the bulk of the codebase. I manually tested the app, maintained the live deployment, added Highlight Hub, and removed Bet Intelligence. I am the sole git committer. I cannot currently explain the full architecture without AI assistance.

Status

Public demo at court-ledger.vercel.app. Auth-gated product flows were not verified live without creating an account during evidence review; public SPA shell and YouTube API were verified.

Technology

React, TypeScript, Vite, Tailwind CSS, Supabase (Auth + PostgreSQL), Vercel serverless API routes, Vitest for limited unit tests. Tesseract.js and SheetJS appear for OCR and export in the repository.

Problem

Personal bet tracking and review without relying on a sportsbook UI — a ledger for history, grading, and simple analytics.

User workflow

Email/password auth gate → authenticated routes for bet entry, history, analytics, markets, Highlight Hub, and settings. Exports and OCR exist in code behind auth. Public /api/youtube-highlights returns highlight metadata without login.

What I personally built

  • Directed AI-assisted development (Cursor + ChatGPT).
  • Manual application testing.
  • Added Highlight Hub.
  • Removed Bet Intelligence (do not treat it as a current feature).
  • Live deployment maintenance.

Architecture or data flow

Browser SPA → Supabase Auth/Postgres; Vercel proxies YouTube highlights; optional edge function for auto-settle exists in repo but is not claimed as live-proven. SQL migrations define intended per-user RLS policies; production RLS application is not independently verified here.

Engineering decisions

Auth-gated shell for personal data; public read endpoint for YouTube highlight metadata; keepalive endpoint fails closed without a cron secret (live 401 without secret observed).

AI-assisted development

ChatGPT for planning, scaffolding, and docs prompts; Cursor for UI generation, debugging, and SQL. The README was AI-generated and is not treated as proof of expertise. Explicit disclosure: I did not independently write or fully understand the underlying codebase.

Quality and verification

Manual app testing by me. Repository includes unit tests for cron-auth helpers (8 tests found in evidence review). No CI workflow runs the test suite. RLS integration tests exist but skip without env and are not claimed as CI-proven.

Edge cases and failure handling

Keepalive rejects unauthenticated cron calls (401 without secret). Auth and data isolation for real users were not exercised live in the evidence pass.

Known limitations

  • Personal demo, not a multi-user product or live product with usage metrics.
  • Bet Intelligence removed — do not advertise.
  • Analytics on the active route use custom CSS bars; Recharts is not claimed as the mounted analytics UI.
  • Market Intelligence controls appear partially non-functional.
  • Notifications icon is presentation-only.
  • Not interview-ready for deep auth/RLS/architecture walkthrough without further study.

Next improvements

Study AuthGate, betsService, and Highlight Hub until explainable without AI; optional authenticated screen recording; verify live RLS if privacy claims are added later.
